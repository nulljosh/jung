Language

Keyword Mapping

Every programming concept maps to Jungian analytical psychology. Both Jungian and standard keywords work interchangeably.

perceive → let Become aware of a value
dream → fn Message from the unconscious
archetype → class Define a universal pattern
emerge → new Arise from the unconscious
Self → this The unified whole
project → print Express to the outside world
manifest → return Make conscious
unconscious → null The unknown
confront → try Face the shadow
embrace → catch Accept what you find
reject → throw Cast into the unconscious
integrate → import Absorb external wisdom
Examples

Code as Psychology

Three examples showing functions, classes, and error handling through the lens of analytical psychology.

hello.jung 
# The individuation process begins here

dream greet(name) {
    project "Hello, " + name
}

greet("World")
project "The journey of individuation begins."
archetypes.jung 
archetype Shadow {
    fn init(name) {
        Self.name = name
        Self.repressed = []
    }

    fn repress(memory) {
        Self.repressed.push(memory)
        project Self.name + " represses: " + memory
    }
}

perceive shadow = emerge Shadow("The Unconscious")
shadow.repress("childhood fear")
shadow.repress("hidden desire")
error_handling.jung 
dream divide(a, b) {
    if b == 0 {
        reject "Cannot divide by the void"
    }
    manifest a / b
}

confront {
    project divide(10, 2)
    project divide(10, 0)
} embrace (e) {
    project "Shadow encountered: " + e
}

Features

Closures, recursion, default parameters. Functions are values.

Classes and Objects

Constructors, methods, Self property access. Archetypes as templates.

Arrays, objects, string interpolation. Built-in methods for each type.

Internals

Architecture

A three-stage pipeline: source text enters the lexer, becomes tokens, flows through the parser into an AST, and is evaluated by a tree-walking interpreter.

Stage 1

Lexer

lexer.c

Tokenizes source into a flat stream. Handles both Jungian and standard keywords, string interpolation, and all operators.

Stage 2

Parser

parser.c

Recursive descent parser. Builds a typed AST with nodes for expressions, statements, classes, and control flow.

Stage 3

Interpreter

interpreter.c

Tree-walking evaluator. Hash table-based scoping, reference-counted values, setjmp/longjmp exception handling.
